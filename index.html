1) Gitflow & Branching Strategy

Ana Branch Yapısı

  • main: Üretim (Production) ortamı. Sadece stabil tag'ler.
  • develop: Ana entegrasyon hattı. Tüm PR'lar buraya gelir.
  • feature/*: Yeni özellik geliştirme. (feature/auth-login)
  • fix/*: Bug düzeltmeleri. (fix/header-mobile)
  • hotfix/*: Canlıdaki acil yangınları söndürme dalı.

Commit Mesaj Standartı

Açık kaynakta Conventional Commits zorunludur:

feat: add go routine for data sync
fix: resolve memory leak in react hook
chore: update docker-compose.yml
docs: update readme instructions

2) Proje Yapısı (Modern Stack)

Golang

Standard Layout

/cmd/app/main.go   # Entry point
/internal/        # Private business logic
/pkg/             # Public helper packages
/api/             # OpenAPI / Swagger
/configs/         # Config templates
.NET / React

Architecture

  • .NET: src/Project.API, src/Project.Core (Clean Architecture).
  • React: /src/components, /src/hooks, /src/store.
  • Docker: Her zaman bir .dockerignore bulundurun.

3) CONTRIBUTING.md (Katkı Süreci)

Önemli: Yeni biri projeye geldiğinde şu adımları izlemelidir:
  1. Projeyi Fork et ve lokalde git clone yap.
  2. Mutlaka bir Issue aç veya var olan bir issue'yu üzerine al.
  3. develop branch'inden yeni bir feature branch aç.
  4. Kodunu yaz, Lint ve Test komutlarını çalıştır.
  5. PR (Pull Request) açarken şablondaki soruların hepsini yanıtla.

4) Donanım & Performans (Low-Spec Dev)

Geliştirme yaparken donanım kısıtlarını aşmak için bu kurallara uyuyoruz:

Docker Kullanımı

Eğer RAM < 16GB ise:

  • Veritabanı (Postgres/Redis) Docker'da çalışsın.
  • Uygulamanın kendisini (Go/Dotnet) native (lokal) çalıştır.
  • docker-compose down --volumes ile kullanılmayan cache'leri temizle.

IDE Optimizasyonu

  • VS Code kullanıyorsan gereksiz eklentileri kapat.
  • Go için gopls bellek kullanımını takip et.
  • Tarayıcıda çok fazla sekme yerine Terminal üzerinden log takibi yap.

Hızlı Başlangıç Komutları

# Bağımlılıkları Kur
go mod download          # Go
dotnet restore           # .NET
npm install --prefers-offline  # React (Hızlı kurulum)

# Çalıştır (Native)
go run cmd/app/main.go
dotnet run --project src/API
npm start
